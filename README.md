# My-First-Project_Amazon-Chess
## I. Overview
This project consisted of two phases: the development of the bot algorithm and the construction of the complete program interface. For the bot algorithm part, the classic minimax algorithm with alpha-beta pruning was implemented, along with certain optimizations to improve efficiency. As for the interface part, due to time constraints and limited personal skill, attempts to create a graphical interface were unsuccessful. Ultimately, a basic character-based interface was adopted, with features such as difficulty selection added.
The following sections provide a brief introduction to the bot algorithm and the program structure.

## II. Bot Algorithm
Due to the inclusion of a difficulty selection feature in the program, two sets of algorithms were developed. Combined with adjusting the search depth, these achieve different difficulty levels.
For the "Easy" difficulty, the most basic random algorithm is used. After enumerating all possible moves for the queen, srand(time(0)) is used to generate a random move, avoiding the repetition of the same move. The main problem with this method is its high randomness; it only guarantees legal moves and is prone to producing strange, self-defeating moves. However, during testing, I occasionally encountered situations where the random algorithm suddenly showed a spark of brilliance and defeated the advanced algorithm.
For the "Medium" and "Hard" difficulties, I used the minimax algorithm with alpha-beta pruning. The difference is that the search depth for "Medium" is 1, while for "Hard" it is 2. In the evaluate function, I used a four-dimensional evaluation function:

- Calculate mobility (number of possible moves). This directly returns the size of the move array.
- Calculate area control. A BFS algorithm is used to calculate the area of the region that the player's pieces can control under the current situation.
- Evaluate piece position. According to the rules of the Amazon game, the closer a piece is to the center, the better its position.
- Evaluate piece flexibility. This returns the number of directions in which the piece can move.
Different weights are assigned to these four dimensions. If a winning or losing move is found, the function directly outputs 99999 or -99999. However, during operation, a problem was discovered: calculating this for every move was too inefficient. Therefore, I first sorted the move array based on the distance from the center and then only calculated the top ten moves, which improved efficiency. Finally, there is the main function part of minimax: if it's the current player's turn, the move with the highest score is selected through recursion; if it's the opponent's turn, the move with the lowest score is selected.
Finally, a check for the legality of moves was added. During testing, it was found that the program sometimes output illegal moves. To avoid this issue, a legality check was incorporated; if the calculated move is illegal, it is regenerated.
Furthermore, due to the strict time limits on Botzone, to enhance performance within the given time frame, I separately optimized the bot algorithm for Botzone. This involved using more precise move generation to avoid redundant calculations and invalid moves; optimizing the evaluation function to reduce computational load; using unordered_map for precomputation and caching to avoid repeated calculations; and optimizing the sorting heuristic for better move ordering. Through these measures, the bot guarantees at least a two-ply search depth within the time constraints.
## III. Main Program Interface Section
The main program interface uses a simple character-based interface. Two different classes, AmazonsGame and GameManager, are used to implement the game's algorithm and interface, and a move structure is introduced to make piece movement more concise. The AmazonsGame class is essentially the same as the bot algorithm code described above, so the following mainly introduces the GameManager class.
The game execution has three main states: MENU, PLAYING, and GAME_OVER. In the menu, you can choose to start a new game, save the game, load a game, or exit the game (i.e., terminate the program). When starting a new game, you can select different difficulty levels. Saving the game stores information such as the current board array, difficulty, and current player into a file named amazon_save.txt in the same directory. Loading a game reads this file to restore the game state. Through these methods, a relatively complete gaming experience is achieved.
